# ระบบอนุมัติเอกสาร (IT-03)

## ภาพรวมระบบ
โปรเจกต์นี้เป็นระบบตัวอย่างสำหรับการจัดการและอนุมัติเอกสาร  
พัฒนาขึ้นเพื่อแสดงความสามารถด้านการออกแบบระบบ การพัฒนา Full Stack  
และการเขียน Unit Test ตามมาตรฐานงานจริง

ระบบรองรับการพิจารณาเอกสาร 3 สถานะ:
- รออนุมัติ
- อนุมัติ
- ไม่อนุมัติ

---

## ความสามารถหลักของระบบ
- แสดงรายการเอกสารทั้งหมด
- เลือกเอกสารที่รออนุมัติได้แบบหลายรายการ (Bulk)
- อนุมัติเอกสาร พร้อมระบุเหตุผล
- ไม่อนุมัติเอกสาร พร้อมระบุเหตุผล
- เอกสารที่อนุมัติแล้วไม่สามารถเลือกซ้ำได้

---

## โครงสร้างระบบ
ระบบถูกออกแบบแบบแยกส่วน (Separation of Concerns)

- Backend: จัดการ Business Logic และข้อมูล
- Frontend: จัดการหน้าจอและประสบการณ์ผู้ใช้งาน
- Testing: มี Unit Test ครอบคลุมฟังก์ชันหลัก

---

## เทคโนโลยีที่ใช้
- Backend: Go (Gin Framework)
- Frontend: React (Vite)
- Testing:
  - Backend: Go testing + Testify
  - Frontend: Vitest + Testing Library + MSW

---

## วิธีใช้งานโดยสรุป
1. เปิดหน้าแสดงรายการเอกสาร
2. เลือกเอกสารที่มีสถานะ “รออนุมัติ”
3. กด “อนุมัติ” หรือ “ไม่อนุมัติ”
4. กรอกเหตุผล และยืนยัน
5. ระบบอัปเดตสถานะเอกสารทันที

---

## หมายเหตุ

- โปรเจกต์นี้มุ่งเน้นการแสดงแนวคิดและคุณภาพโค้ด
